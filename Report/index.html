Sprint 1 (October 23 - November 2)

Sprint Completed Stories

  • Set Up Git Hub
  • Use Bootstrapping for HTML/CSS
  • Header and Footer on Homepage
  • Names in the About Page
  • HTML Template Report, Separate Folder for version control and Report App

This week we created our team assignment, familiarized ourselves with the team github page (as well as creating and deploying on Heroku). We set up team roles (with Mallika Bharadwaj as Product Owner, Gopikrishnan as Scrum Master and Team Member and Chris Chen as Lead Developer), and estimated the story point values of our stories and upcoming product backlog. Although we did not have any conflicts when merging different parts of our code, we did have to assign product owner roles, and establish different tasks for each role. With these assigned, the team will better be able to react to changing requirements from Professor Jensen (the Shareholder(s)).

Despite our original story point estimations, we realized mid-way through (during and after day 4) that implementing the HTML template for our reports and version control was taking longer than we expected. It was a difficult task to locate the best place to create such a template, and integrate it into our code base. So, this increased our story point estimate for the story.
